Client–Server Architecture

Figure: Client–Server Architecture

Client–Server architecture is a network model in which a client sends a request and the server provides a response. It is commonly used in web applications.

Trace What Happens When You Open a Website

  1. User enters the URL in browser.
  2. Browser checks DNS to find IP address.
  3. Browser sends HTTP request to server.
  4. Server processes the request.
  5. Server sends HTML, CSS, JS files.
  6. Browser renders and displays the webpage.

Conclusion

Thus, client–server architecture allows communication between users and web servers through request and response mechanism.
